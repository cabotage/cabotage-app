Connect this organization to your Tailscale network. Services with Tailscale ingresses will be accessible on your tailnet.

Setup Instructions
{# Step 1: Create Tags #}

1. Create ACL Tags

In your tailnet policy file, add two tags. {{ copyable('tag:' ~ tp ~ '-operator') }} is used by the platform to manage devices. {{ copyable('tag:' ~ tp) }} is applied to the devices that serve your ingresses. The operator tag must own the service tag. 

"tagOwners": {
  "tag:{{ tp }}-operator": [],
  "tag:{{ tp }}": ["tag:{{ tp }}-operator"]
}
{# Step 2: Setup ACLs #}

2. Configure ACL Policies

Allow devices to advertise Tailscale Services without manual approval, and enable Funnel for public internet access.

Add to your tailnet policy JSON:

"autoApprovers": {
  "services": {
    "tag:{{ tp }}": ["tag:{{ tp }}"]
  }
},
"nodeAttrs": [
  {
    "target": ["tag:{{ tp }}"],
    "attr": ["funnel"]
  }
]
  1. Go to Access Controls > Auto Approvers
  2. Under Services, add: Service {{ copyable('tag:' ~ tp) }} can be auto-approved by {{ copyable('tag:' ~ tp) }}
  3. Go to Access Controls > Node Attributes
  4. Add: Target {{ copyable('tag:' ~ tp) }}, Attribute {{ copyable('funnel') }}
{# Step 3: Enable HTTPS #}

3. Enable HTTPS Certificates

Go to DNS and enable HTTPS Certificates. This allows ingress proxies to obtain TLS certificates for your tailnet domain.

{# Step 4: Trust Credentials #}

4. Create Trust Credential

Go to Tailscale Admin > Trust Credentials and click Credential.

4a Settings
  1. Select OpenID Connect
  2. Set Issuer to Custom issuer
  3. Issuer URL: {{ copyable(oidc_issuer_url) }}
  4. Subject: {{ copyable('org:' ~ organization.k8s_identifier) }}
  5. Leave Audience and Custom claims blank
  6. Click Continue
4b Scopes
  1. Set Scopes to Custom scopes
  2. Under General > Services: check Write
  3. Under Devices > Core: check Write
  4. Under Keys > Auth Keys: check Write
  5. Set Tag to {{ copyable('tag:' ~ tp ~ '-operator') }}
  6. Click Generate credential
