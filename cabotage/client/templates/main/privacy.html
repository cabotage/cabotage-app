The short version

We collect only what's necessary to operate the platform -- account info, deployment metadata, and access logs. No data sales, no third-party sharing.

{# §1 Overview #}

1 Overview

This Privacy Policy describes how the Cabotage platform ("Service") collects, uses, and protects information when you use our Platform as a Service.

{# §2 What We Collect #}

2 What We Collect

{{ icon('user', 'w-4 h-4') }}

Account Information

Username, email address, authentication credentials (hashed), and organization membership.

{{ icon('container', 'w-4 h-4') }}

Deployment Data

Application configuration, environment variables (encrypted), container images, build logs, and deployment history.

{{ icon('activity', 'w-4 h-4') }}

Usage & Access Logs

IP addresses, request timestamps, browser user-agent strings, and actions performed within the platform.

{{ icon('git-branch', 'w-4 h-4') }}

Source Integration

Repository URLs, branch names, and webhook payloads used to trigger automated builds and deployments.

{# §3 How We Use Data #}

3 How We Use Data

  • {{ icon('check', 'w-4 h-4') }} Authenticate your identity and manage access control
  • {{ icon('check', 'w-4 h-4') }} Build, deploy, and manage your applications
  • {{ icon('check', 'w-4 h-4') }} Provide audit trails and operational logging
  • {{ icon('check', 'w-4 h-4') }} Troubleshoot issues and improve platform reliability
  • {{ icon('check', 'w-4 h-4') }} Enforce acceptable use policies and protect infrastructure
{# §4 Data Storage #}

4 Data Storage

All data -- including application code, configuration, logs, and database contents -- is stored on infrastructure operated by Cabotage. For self-hosted deployments, your organization's administrator controls where data is stored and processed.

Sensitive values such as environment variables and registry credentials are encrypted at rest.

{# §5 Data Sharing #}

5 Data Sharing

{{ icon('shield-check', 'w-4 h-4') }}

We do not sell, trade, or share your data with third parties.

Data may be shared with third-party services only when necessary to operate the Service (e.g., container registries, Git hosting providers) or when explicitly configured by you or your organization.

{# §6 Security #}

6 Security

  • {{ icon('lock', 'w-4 h-4') }} All credentials and secrets are encrypted at rest
  • {{ icon('shield', 'w-4 h-4') }} HTTPS/TLS is enforced for all web traffic
  • {{ icon('key', 'w-4 h-4') }} Two-factor authentication is available for all accounts
  • {{ icon('users', 'w-4 h-4') }} Role-based access control limits data exposure
  • {{ icon('file-text', 'w-4 h-4') }} Audit logging tracks all significant platform actions
{# §7 Retention #}

7 Retention

Account data is retained as long as your account is active. Deployment metadata and build logs are retained according to the platform's retention policies.

Upon account deletion, personal data is removed. Anonymized audit logs may be retained for security and compliance purposes.
