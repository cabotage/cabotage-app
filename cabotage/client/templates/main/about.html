{# §1 What is Cabotage #}

What is Cabotage

Cabotage is a platform that handles the operational work of running applications — container builds, deployments, TLS, secret management, scaling, and more — so your team can focus on shipping code.

Built by Cabotage Deploys, LLC, the platform runs on Kubernetes and is designed for teams that want the convenience of a managed PaaS without the overhead of building deployment tooling in-house.

{# §2 What you can do #}

What you can do

{{ icon('git-branch', 'w-4 h-4') }}

Push to deploy

Connect a GitHub repository and deploy on push. Cabotage builds your container image and rolls it out.

{{ icon('layers', 'w-4 h-4') }}

Multi-process apps

Run web servers, background workers, scheduled jobs, and one-off tasks from a single codebase with a Procfile.

{{ icon('lock', 'w-4 h-4') }}

Secrets and config

Manage environment variables and secrets through the dashboard. Values are stored in Vault and injected at runtime.

{{ icon('shield', 'w-4 h-4') }}

Automatic TLS

TLS certificates are provisioned and renewed automatically. Traffic is terminated at the pod level.

{{ icon('users', 'w-4 h-4') }}

Organizations and teams

Group projects under organizations. Control who can deploy what.

{{ icon('activity', 'w-4 h-4') }}

Logs and metrics

View build and runtime logs from the dashboard. Optionally forward metrics and traces to Datadog.

{# §3 How it works #}

How it works

  1. 1

    Connect your repository

    Link a GitHub repo to a Cabotage application. Define your processes in a Procfile.

  2. 2

    Cabotage builds and deploys

    Each push triggers an image build. When the build completes, Cabotage creates a rolling deployment on your cluster.

  3. 3

    Manage from the dashboard

    Scale processes, update configuration, view logs, and roll back — all from the web interface.
